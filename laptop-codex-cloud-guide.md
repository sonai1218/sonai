# 집 노트북에서 Codex Cloud 이어서 쓰는 법

이 문서는 집에 가서 노트북으로 Codex Cloud를 다시 사용할 때 보는 안내서입니다.
어렵게 생각하지 말고, 아래 순서대로만 하면 됩니다.

## 1. 먼저 열어야 하는 곳

노트북에서 인터넷 브라우저를 열고 아래 주소로 들어갑니다.

https://chatgpt.com/codex

ChatGPT 계정으로 로그인합니다.

## 2. GitHub 저장소 선택

Codex 화면에서 GitHub 저장소를 고르라고 나오면 아래 저장소를 선택합니다.

`sonai1218/sonai`

이 저장소는 소나이 관련 Codex Cloud 작업을 위한 기본 작업방입니다.

## 3. 작업 시작 전에 붙여넣을 문장

새 작업을 시작할 때 아래 문장을 그대로 복사해서 붙여넣습니다.

```text
AGENTS.md와 context.md를 먼저 읽고, 그 기준에 맞춰 작업해줘.
나는 비개발자라서 결과물은 쉽게 수정할 수 있는 구조로 만들어줘.
가능하면 HTML, CSS, JavaScript 기반의 단순한 파일로 만들어주고, 고객정보나 민감한 자료는 GitHub에 올리지 말아줘.
작업이 끝나면 무엇을 만들었는지, 어떤 파일을 바꾸면 되는지, 어떻게 열면 되는지 쉽게 설명해줘.
```

## 4. 이렇게 말하면 됩니다

예시 1:

```text
AGENTS.md와 context.md를 먼저 읽고, 그 기준에 맞춰 작업해줘.
소나이 상품 상세페이지에 쓸 수 있는 간단한 HTML 템플릿을 만들어줘.
```

예시 2:

```text
AGENTS.md와 context.md를 먼저 읽고, 그 기준에 맞춰 작업해줘.
Settlemier's Jackets 커스텀 오더 안내 페이지를 단일 HTML 파일로 만들어줘.
```

예시 3:

```text
AGENTS.md와 context.md를 먼저 읽고, 그 기준에 맞춰 작업해줘.
이미지 경로와 문구를 나중에 내가 쉽게 바꿀 수 있게 주석을 달아줘.
```

## 5. 하면 안 되는 것

아래 자료는 public GitHub 저장소에 올리면 안 됩니다.

- 고객 이름
- 전화번호
- 주소
- 주문서 원본
- 매출 자료
- 계정 비밀번호
- API 키
- 내부 정산 자료

이런 자료가 필요한 작업은 먼저 개인정보를 지우거나, 샘플 파일로 바꿔서 써야 합니다.

## 6. 결과가 나오면 확인할 것

Codex가 작업을 끝내면 바로 믿고 쓰지 말고 아래만 확인합니다.

- 어떤 파일이 만들어졌는지
- 내가 바꿔야 할 문구 위치가 설명되어 있는지
- 이미지 경로를 어디서 바꾸는지
- 고객정보가 들어가 있지 않은지
- 너무 복잡한 설치 과정이 생기지 않았는지

소나이는 1인 운영 기준이 중요합니다. 보기 좋아도 관리가 어려우면 좋은 결과물이 아닙니다.

## 7. 헷갈릴 때 다시 붙여넣을 문장

뭔가 복잡해지면 아래 문장을 다시 보냅니다.

```text
너무 복잡합니다. 소나이 1인 운영 기준으로 더 단순하게 줄여주세요.
외부 라이브러리는 꼭 필요한 경우만 쓰고, 내가 나중에 텍스트와 이미지 경로를 쉽게 바꿀 수 있게 해주세요.
```

## 8. 기억할 것

Codex Cloud는 내 컴퓨터 안의 채팅창을 그대로 옮기는 기능이 아닙니다.
대신 GitHub 저장소 안의 `AGENTS.md`와 `context.md`를 읽고, 그 기준에 맞춰 새 작업을 이어가게 하는 방식입니다.

그러니까 새 노트북에서 시작할 때는 항상 이 한 문장만 기억하면 됩니다.

```text
AGENTS.md와 context.md를 먼저 읽고, 그 기준에 맞춰 작업해줘.
```
